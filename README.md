# agents.md

This repository exists to keep a canonical `AGENTS.md` file and preserve its history in git.

## What Lives Here

- `AGENTS.md`: the primary artifact and operating contract
- small supporting docs only when they clearly improve maintainability

The goal is to keep this repo intentionally minimal. History should read like the evolution of an operating manual, not a general-purpose project.

## Conventions

- `AGENTS.md` is the canonical file name for tool compatibility
- changes should stay small, reviewable, and easy to verify
- commit messages should explain why a change to the operating document matters
